    • ACTIONS:
      • OnPAKChange

        typedef struct
        {

          DWORD Code;
          DWORD ID;
          char *Name;
          char *Author;
          char *Help;
          HANDLE hInst;

          PLUGPROC ReceiveActions
          PLUGPROC OnPAKChange
          DWORD Reserved[3];

        } TPlugInfo, *LPPlugInfo;


        The OnPAKChange Field of the TPlugInfo can be set to a local-PLUGPROC-fonction to handle changes in the PAKs files; When a PAK File is modified, Dragoon's checks the existence of the OnPAKChange fonction and then Call it with in first parameter the FullName of the PAKFile ( including fullpath ). The second parameter is always NULL.

        What to Do ?
        What you want.... PAK may have be modified, or simply saved....

        What does it return ?
        what you want, Dragoon's is not handling errors for this fonction.
