2D library :

2D bitmap rendering : to display 2D sprites, two classes are available : Sprite and Group. Sprite allow you to draw a bitmap at a precise point and to change its size, color and position. Group allow to gather many Sprites, draw them in a "depth" determined order and make them rotate, resize or other transformations.

This library is part of Standard (Std) library, so it is automatically included in all MTS files.

Group :

Two groups types are present : principal groups and sub-groups. A prncipal group is created with a specified z value (between 0 and 1). It will allow to display a group of Sprites each with the same z value in order to intercalate many Sprites planes with 3D objects (this ones are drawn behind or in front of Sprites planes depending of their z) value.

On the other hand, a sub-group simply allow to gather Sprites to transform them simultaneously. A sub-group must belong to a principal group or another sub-group.

In a same group, Sprites and sub-groups are sorted by "depht" (drawing priority), those with tallest depth being first drawn. Many Sprites and sub-groups can have the same depth, in this case drawing priority is given to the last added.
new Group(z) create a new principal group with z value.
new Group(g) create a new sub-group and add it to g.
_x read or write x group postition.
_y read or write y group postition.
_visible allow to know if the group is visible and to change its visibility.
_depth read or write group drawing priority.
_color read or write group color mask.
_alpha read or write group transparency coeficient (between 0 and 1).
_parent (read only) : return the sub-group parent group or null if it is a principal group.
remove() destroy the group, all its sub-groups and sprites added to.
_xscale read or write horizontal scale factor.
_yscale read or write vertical scale factor.
_rotation read or write group rotation angle (radian) : all sub-elements of the group will do a rotation refering to group (0,0) point.
_z read or write z value of a principal group.

Sprite :

A Sprite allow to draw a bitmap picture at a given site.
new Sprite(g) create a new sprite and add it to g group.
_x read or write x sprite coordinate.
_y read or write y sprite coordinate.
_visible allow to know if the sprite is visible and to change its visibility.
_depth read or write sprite drawing priority.
_color read or write sprite color mask.
_alpha read or write sprite transparency coeficient (between 0 and 1).
_parent (read only) : return sprite parent group.
remove() destroy the sprite.
_width read or write horizontal sprite size.
_height read or write vertical sprite size.
_tu _tv _tu2 _tv2 read or write sprite textures coordinate (see below).
_bitmap read or write sprite picture used. (null or PIC resource type)

Picture displayed by the sprite correspond to a part of its _bitmap picture. To select this part, we use textures coordinates (tu,tv) and (tu2,tv2), between 0 and 1. So (0,0)-(1,1) correspond to the full picture and (0.5,0)-(1,0.5) correspond to the upper right picture quarter.

Text :

A text is displayed as a sprite, it must be included in a group.
new Text(g) create a new text and add it to g group.
_x read or write x text coordinate.
_y read or write y text coordinate.
_visible allow to know if the text is visible and to change its visibility.
_depth read or write text drawing priority.
_color read or write text color mask.
_alpha read or write text transparency coeficient (between 0 and 1).
_parent (read only) : return text parent group.
remove() destroy the text.
_text read or write displayed string.
_font read or modify font used to display text. null value make the text invisible. Default font is null.
